# Code Review Skill

当你进行代码审查或准备提交改动时，请遵循以下规范：

## 1. 代码审查清单
- **正确性**: 逻辑是否正确？是否处理了边界情况？
- **可读性**: 变量名是否清晰？逻辑复杂处是否有注释？
- **安全性**: 是否有硬编码的密钥或敏感信息？

## 2. Git Commit 规范 (重要：防止 GitHub 乱码)
- **语言**: 所有的 commit 消息必须使用 **中文 (简体中文)**。
- **格式**: 采用 Conventional Commits 规范。
  - 格式: `type(scope): 描述`
  - 常用类型: `feat` (新功能), `fix` (修复), `docs` (文档), `style` (格式), `refactor` (重构), `test` (测试), `chore` (构建/工具)。
  - 示例: `feat(ui): 增加用户头像上传功能`
- **编码**: 必须确保输出为 **UTF-8** 编码。在生成 Git 命令时，确保描述部分不包含导致乱码的特殊字符。

## 3. GitHub 兼容性
- 始终以 UTF-8 字符集思考和输出。
- 在审查结束后，如果用户要求提交，请按照上述规范生成中文 commit message。
